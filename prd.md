# 4.1 计时系统（核心模块）

---

## 4.1.1 功能说明

系统支持两种计时模式：

### 模式 A：正计时（Count Up）

用户点击开始 → 系统记录开始时间 → 持续累积时长 → 点击结束生成记录

---

### 模式 B：倒计时（Count Down）

用户输入目标时间 → 系统倒计时 → 时间结束自动停止并记录

---

## 4.1.2 状态机设计

```text
IDLE → RUNNING → PAUSED → FINISHED
```

---

## 4.1.3 状态说明

| 状态     | 描述     |
| -------- | -------- |
| idle     | 未开始   |
| running  | 正在计时 |
| paused   | 暂停     |
| finished | 已结束   |

---

## 4.1.4 用户操作

| 操作   | 行为     |
| ------ | -------- |
| Start  | 开始计时 |
| Pause  | 暂停     |
| Resume | 恢复     |
| Stop   | 结束     |

---

## 4.1.5 UI 要求

- 显示当前时间（HH:mm:ss）
- 显示当前分类
- 显示计时模式（正/倒）
- 按钮状态随状态变化

---

## 4.1.6 关键逻辑（必须实现）

### ❗ 不允许使用 setInterval 作为唯一计时来源

必须使用：

```ts
duration = currentTime - startTime + accumulatedPausedTime;
```

原因：

- 防止 App 切后台导致时间错误

---

## 4.1.7 边界情况

- App 被杀死 → 恢复计时状态
- 切后台 → 时间仍正确
- 倒计时结束时：
  - 自动停止
  - 触发通知
  - 生成记录

---

## 4.1.8 数据结构

```ts
interface TimeRecord {
  id: string;
  userId: string;
  subjectId: string;
  mode: "countup" | "countdown";
  duration: number;
  startTime: number;
  endTime: number;
  pauses: PauseRecord[];
  createdAt: number;
}

interface PauseRecord {
  start: number;
  end: number;
}
```
